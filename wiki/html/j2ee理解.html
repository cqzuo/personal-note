J2EE的概念

  • 是一种利用java2平台来简化企业解决方案的开放部署和管理相关复杂问题的体系结构
  • 巩固了标准版的优点(可移植性,方便存取数据库的jdbc,corba以及在internate应用中保护数据的安全模式)
  • 提供EJB JAVA SERVLET JSP 以及xml技术的全面支持

J2EE的优势

保留现存it资产

  • 以渐进方式建立在已有系统之上

高效开发

  • 将通用和繁琐的服务端任务交给中间件供应商完成

状态管理服务

  • 开发人员不用关心状态管理

持续性服务

  • 开发人员不用对数据访问逻辑进行编码就能编写应用程序,易于开发维护

分布式共享数据对象cache服务

  • 让开发人员能开发高性能的系统,极大提高整体部署的伸缩性

支持异构环境

  • j2ee能够开发部署在异构环境的可移植程序,不依赖于任何特定操作系统,中间件,硬件

可伸缩性

  • 基于j2ee平台的应用程序能被部署到各种操作系统上
  • j2ee领域的供应商提供了更为广泛的负载平衡策略,能消除系统中的瓶颈,允许多台服务器集成部署

稳定的可用性

  • j2ee部署到可靠的操作环境中,支持长期的可用性

四层模型

介绍

  • j2ee使用多层的分布式应用模型,将c/s模型中的不同层平分成多层

j2ee应用程序组件

  • j2ee应用程序由组件构成,组件是具有独立功能的软件单元
  • 应用客户端程序和applets是客户层组件
  • Servlet和jsp是web层组件
  • ejb是业务层组件

客户层与web层

  • 基于web或传统模式
  • jsp/servlet html和java applets不算web层组件
  • 包含某些javaBean对象来处理用户输入,并将输入发送给业务层来处理

业务逻辑层组件

  • 业务层代码的逻辑用来满足客户特定需求,有ejb进行处理
    • sessonBean 与客户端程序的临时交互
    • entityBean 数据库的表中一行永久的记录
    • messagedrivenBean 结合sessonBean和JMS消息监听器的特性,运行业务层组件异步接收JMS消息

企业信息系统

  • 企业基础建设系统(erp),大型机事务处理 数据库系统 其他信息系统

j2ee的框架

优点

  • 业务逻辑被封装为可复用的组件
  • 服务器以容器的形式为所有的组件类型提供后台服务

服务

j2ee安全模型

  • 配置web组件或enterprise bean,让授权用户才能访问资源
  • 给客户分配角色,只允许激活特定方法
    • enterprise bean的部署描述中声明角色和激活方法

事务管理

  • 可以指定组成一个事务中所有方法间的关系,同一事务的方法被作为一个单一单元
  • 在部署描述文件中声明enterprise bean的事务属性,并处理相应事务

jndi寻址

  • 为企业内多重名字和目录服务提供了一个统一的接口

j2ee远程连接

  • 管理客户端和enterprise bean间的底层交互

生存周期管理

  • 管理enterprise bean的创建和移除

数据库连接池

容器

  • ejb容器管理所有J2ee应用程序企业级bean的执行
    • enterprise bean和它们的容器运行在j2ee服务器上
  • web容器管理jsp和servlet组件的执行
    • web组件和他们的容器运行在j2ee服务器上
  • 应用程序客户端管理所有j2ee 应用程序中客户端组件的执行
    • 应用程序客户端和他们的容器运行在j2ee服务器上
  • applet容器运行在客户端机器上的web浏览器

核心技术

  • j2ee平台由一整套服务,应用程序接口和协议组成
  • jdbc
    • 对数据库的访问具有平台无关性
  • jndi
    • 执行名字和目录服务,提供了统一的模型来操作本地资源和服务器中的对象
  • ejb
    • 提供了一个框架来开发和部署分布式的商务逻辑,简化了企业级应用开发的难度
  • rmi
    • 使用序列化方式在客户端和服务器端传递数据
  • java Idl/corba
  • jsp
  • servlet
  • xml
  • jms
  • jta
  • jts
  • java mail

设计观念

Professinal

  • j2ee应用有各个组件组成,每个组件领域的开发者都应该是专业的

Configruation

  • 可配置性是ejb重用的基础

Simple

  • 将大问题细化,分而治之

Reusable

  • 可重用

Communication

  • 开发者之间,开发者与客户之间的沟通
